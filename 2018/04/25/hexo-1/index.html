【第一篇】使用Hexo + github pages搭建个人博客

发表于 2018-04-25 | 分类于 技术
  • Github 是开源代码仓库，github pages 是它的一个功能，允许你上传网页文件，并分配一个域名供访问
  • Hexo 是一个静态博客生成工具，支持使用markdown语法

理论上不需要hexo也可以，也可以使用github pages写博客。

使用hexo的好处有下面几点：

  • 现成的博客框架
  • 美观的主题，想换就换

同样的轻量博客框架还有jekyll，对比过jekyll和hexo，在windows 10下jekyll需要安装ruby环境，官方也不推荐在windows上使用，而hexo是node环境就可以，所以选择hexo。

hexo官网

我使用的hexo版本是”3.2.0”

常用命令

hexo new 新文章名 // 新增文章
hexo s // server开启本地调试
hexo g -d // 生成并发布

更换主题

git clone ${主题git地址} themes/${主题名}

如： git clone git@github.com:theme-next/hexo-theme-next.git themes/next

修改_config.yml文件为：

theme: next

插入图片 控制图片大小

推荐使用 img标签 外部图片 大小可控

<img src="https://www.zoivi.cn/image/5.jpg" width="300" height="210" />

推荐使用 img标签 存放在source/images下 大小可控

<img src="/images/baobao.jpg" width="280" height="220" alt="baobao"/>

下面三种看个人需要 配置文件_config.yml要先设置ost_asset_folder: true，然后hexo new 一个新文章，会有一个同名文件存放如图片

img标签 存入在对应文章图片文件夹中 大小可控 (hexo生成的相对路径不准确，比如首页无法显示，在文章详情页可以正常显示，)

内部引用 存入在对应文章图片文件夹中 大小不可控

markdown语法 大小不可控 存放dm.jpg在source/images下![logo](/images/dm.jpg)

使用个人域名绑定github pages

  1. 先买个域名
  2. ping一下github原来分配给你的域名 如：harryqq.github.io

(为什么是两个q？因为当初github账号一个q的被注册了)

  1. 解析到ip,并做www的指向
  2. 设置项目的Custom domain，如harryq.c
  3. 稍等片刻（一般五分钟内可以） 打开你的域名，看是否成功

预览摘要功能

默认首页的文章也全部展开，这个在next主题下_config.yml文件将auto_excerpt设为true即可。
