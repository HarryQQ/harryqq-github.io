面向过程

结构化程序的弊端：
1.缺乏对数据的封装，（如对变量的封装，）
2.数据和方法（对数据的操作）的分离;

面向对象

类：具有相同特征和同共行为的一类（东西），是对象的模板
跟在class后面的是类名，可以表示一个类
对象：实际存在的一个个体（实例）

对象的创建

类 引用变量=new 类名();//类名() 就是该类的无参构造器//new 类名(）–创建的对象
类 引用变量=new 类名(参数列表);//类名(参数列表) 就是该类的有参构造器

例如：
控制台输出
Scanner sca=new Scanner(System.in);
随机数
Random ran=new Random();
窗体对象
JFrame frame=new JFrame();
创建数组对象
int[] num=new int[元素个数];//char[] chs=new char[26]//boolean[] flag=new boolean[26];

类设计：

1.先把到这一类（东西）里面的多个个体，进行个体属性（特征）的分析（抽取数据）。
2.分析后，找一些有实际意义共同的名字，来对这些属性进行描述和定义。
3.再分析这一类（东西）拥有的共行的行为：把行为定义为方法;

不是基本类型的都是引用型，引用型数据默认值都是null
非基本类型称为引用 类型
引用类型生成的变量称为引用类型变量，简称为引用

  1. 现实世界中由很多对象组成的
  2. 现实世界中，先有对象，基于对象抽出了类。（类型）
    代码中，是先确定对象的类型，再创建对象。
    以后写程序：1.要先找到对象后 2.再给对象抽类
  3. 类是一个模子，对象是类的一个实例
  4. 类中可以包含：
    1）所有对象所共有的特征（静的）属性
    2）所有对象共有的行为(动的) 方法
  5. 一个类可以创建很多对象
    同一个类创建的对象，结构相同，数据不同。

方法的签名

方法的签名=方法名+参数列表(参数的类型，与参数名无关)。
同一个类中，不允许有两个签名完全一样的方法存在。
如果一个类的两个方法只是方法名相同而参数列表不同，是可以的。
在Java语言中，允许多个方法的名称相同，但参数列表不同，此种方式称为方法的重载(overload)

下面这个也是方法重载的一个例子

System.out.println("Hello world!");
System.out.println(24235.53);

public static void say(){};
public static void say(String name){};//可以
public static void say(String name,int age){};//可以
public static void say(String address){};//不可以，因为类型跟上面的一样，与参数名字无关
public static void say(int age,String name){};//可以，因为参数类型的位置顺序不一样，再强调一次，跟参数名称无关。

编译时根据签名绑定调用方法。
可以把方法重载看成不同的方法，刚好名称是一样的。

public static void say(String name,String address){};
public static void say(String name,double salary){};

构造方法

是类中定义的方法，不同于其它的方法，构造方法定义有如下规则：

  • 与类同名，没有返回值，但是不能写void
  • 常常用于给成员对象赋初始值
  • 构造方法是在创建对象（new）时初自动调用
  • 若自己不写构造方法，则编译器默认（default）提供一个无参的（自己写了，默认无参的就不给了）
  • 一般有多个构造方法，即是用到重载，也要自己加一个默认的。

构造方法可以重载

  • this指的是当前对象.哪个对象创建了，类里面方法的this都是指这个对象。
  • this的用法：
  • this.成员变量—-访问成员变量
  • this.方法———调用方法
  • this()————调用构造方法

对象内存管理

用new创建的对象储存在堆内存中

当一个对象没有任何引用，视为废弃对象，属于被回收的范围，该对象的成员变量也随之被回收。
成员变量的生命周期：从对象在堆中创建开始到从堆中被回收结束。

GC（Garbage Collection）垃圾回收器，JVM自带的一个线程（自动运行的程序）

（有一个成员变量，没有用了，但是还有引用，所以不会被GC回收。）
内存泄漏指，不再使用的内存没有及时被回收，严重的内存泄漏会导致程序的崩溃。

System.gc()方法用于建议虚拟机马上调度GC线程回收资源。（只是建议快点来回收）具体什么时候实施策略取决于不同的JVM系统

栈空间用于储存方法中的局部变量
一个栈桢对应一个正在调用的方法，储存了该方法的参数，局部变量等数据。
方法调用完后，其对应的栈桢将被清除。
局部变量失效。

继承extends

Sub Class(子类) 可以继承 Super Class(父类)的成员变量及方法，也可以定义自己的成员变量和成员方法。
java语言不支持多重继承.

Sub Class通过super()调用Super Class构造方法//必须放在子类构造方法的第一行
如果子类没有调用父类的构造方法，java编译器会自动加上对父类的无参构造方法的调用。（如果该父类没有无参构造方法，会有编译错误）

父类的引用可以指向子类的对象,但是只能访问父类所定义的成员，不能访问子类自己扩展的部分

向上造型：
一个父类型的引用指向一个子类的对象，是Java多态的体现,一个子类对象可以向上造型为父类的类型

继承具有传递性

重写(override)：

override的两同两小一大原则：

  • 两同：方法名相同，参数列表相同
  • 两小：子类返回值类型小于等于父类的，子类抛出的异常小于等于父类的。
    • void和基本类型的必须是同样的类型
    • 引用类型，可以子类小于父类，父类大，子类小
  • 一大：子类方法的访问权限要大于等于父类的

重载（overload）和重写（override）的区别：

- override:发生在两个类中（父子类关系），方法相同，参数列表不同 “运行期绑定”
- overload:发生在同一个类中，方法签名相同“编译期绑定”

抽象类的意义：

  1. 封装子类共有的变量和方法–代码的重用
  2. 为子类提供一种公共的类型–向上造型
  3. 定义抽象方法，子类分别重写它，为子类提供一个公共入口

抽象方法

  • 由abstract修饰
  • 只有方法的定义，没有方法的实现（没有方法体）
  • 不完整的方法

抽象类

  • 包含抽象方法的类，必须是抽象类
  • 由abstract修饰，不能被实例化（因为方法不完整，类也不完整，所以不能new对象）
  • 抽象类需要被继承的（没有不能实例化，没有意义，所以必须被继承）

子类继承后：

  • 一般override抽像方法
  • 也可以继承抽象方法

接口

  • 一个标准，规范
  • 只要遵守了标准，就能干某件事
  • 由interface定义 是一种特殊的抽象类,接口中只能包含常量和抽象方法,数据默认为常量，方法默认为抽象 可以被忽略的修符
  • 接口需要被实现，子类实现接口，必须重写接口中的所有抽象方法
  • 一个类可以实现多个接口
  • 当又要继承一个类，又要实现一个接口时，先继承再实现接口
  • 接口之间可以继承（extends）
