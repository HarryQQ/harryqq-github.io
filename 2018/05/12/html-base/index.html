行内标签和块级标签

行内元素不会以新行开始，而块级元素会新起一行

行内元素：span,img,a,input,textarea,button,select,label

块级元素：div,p,hr,h1-6,ul,table,audio,video

HTML5新特性，语义化

  1. 多媒体播放：video 影像和audio 音频元素
  2. 对本地离线存储的更好的支持(localStorge和sessionStorge )
  3. 新的特殊内容元素，比如 article、footer、header、nav、section
  4. 新的表单控件，比如 calendar、date、time、email、url、search
  5. 用于绘画的 canvas 元素

使用data-的好处

在所有 HTML 元素上嵌入自定义 data 属性的能力,值是一个字符串 

var userEl = document.querySelector('#user')
console.log(userEl.dataset.userName) // qiu
userEl.dataset.phone = '13631256680'
console.log(userEl.dataset.phone) // 13631256680

dataset目前浏览器支持不多，建议先用
userEl.setAttribute('age', 25);       
userEl.log(userEl.getAttribute('data-age')); //25

从用户输入浏览器输入url到页面最后呈现，有哪些过程

参考

请求资源

  1. 用户输入URL地址
  2. 浏览器解析URL解析出主机名
  3. 浏览器将主机名转换成服务器ip地址（浏览器先查找本地DNS缓存列表 没有的话 再向浏览器默认的DNS服务器发送查询请求 同时缓存）
  4. 浏览器将端口号从URL中解析出来
  5. 浏览器建立一条与目标Web服务器的TCP连接（三次握手）
  6. 浏览器向服务器发送一条HTTP请求报文
  7. 服务器向浏览器返回一条HTTP响应报文
  8. 关闭连接 浏览器解析文档
  9. 如果文档中有资源 重复6 7 8 动作 直至资源全部加载完毕

渲染页面

  1. HTML解析出DOM Tree
  2. CSS解析出Style Rules
  3. 将二者关联生成Render Tree
  4. Layout 根据Render Tree计算每个节点的信息
  5. Painting 根据计算好的信息绘制整个页面

js和css存放位置

html,js,css总的来说就是按照html文档的顺序加载,

文档解析过程，碰到script标签会立即解析脚本，停止解析文档（因为JS可能会改动DOM和CSS，所以继续解析会造成浪费）。最好把js放在body结束标签前，如果放在head，页面需要等待所有js文件加载完，才往下执行，造成页面白屏

如果脚本是外部的，会等待脚本下载完毕，再继续解析文档。现在可以在script标签上增加属性 defer或者async。

脚本解析会将脚本中改变DOM和CSS的地方分别解析出来，追加到DOM Tree和Style Rules上。

问题处理：

vue学习例子中，引用框架js后，再引用index.js,中间是对id为app的绑定一个data,如果script放在head中，由于html还没有被渲染，所以不起作用。

解决办法一：是对在index.js中加一个 

window.onload = function() { // xxxx方法 }

解决办法二：将标签放于body标签后

前后端渲染区别

前端渲染
spa页面基本都是基本由js渲染出来，搜索引擎爬虫无法识别js。

1、用ajax进行局部更新，用户体验好。

2、前后分离，服务器用来访问数据库提供接口，前端通过接口获取数据和展示数据。

3、节约服务器成本。

后端渲染

1、首页性能（单页面应用等待下载js资源多，用户网络差的情况下首页加载时间会很长）
2、利于seo,服务器返回的渲染好的html页面。

html模版语言

后端板模语言：jsp和php，类似html写法，又能使内容的动态化
前端模板语言：vue提供模板语法，底层的实现上，Vue 将模板编译成虚拟 DOM 渲染函数。结合响应系统，Vue 能够智能地计算出最少需要重新渲染多少组件，并把 DOM 操作次数减到最少。如果熟悉虚拟dom和喜欢js原生，可以直接用jsx写。

meta viewport原理

meta viewport原理

<meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
  • width：控制 viewport 的大小,可以具体值比600,device-width为设备的宽度
  • height：与 width 相对应，指定viewport 高度。
  • initial-scale：初始缩放比例，也即是当页面第一次 load 的时候缩放比例。
  • maximum-scale：允许用户缩放到的最大比例。
  • minimum-scale：允许用户缩放到的最小比例。
  • user-scalable：是否允许用户手动缩放。
