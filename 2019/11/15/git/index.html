git常用操作

发表于 2019-11-15

常用

  • git config –list // 查看用户名、邮箱等配置
  • git config –global user.name // 配置git用户名

  • git config –global user.email // 配置邮箱

  • git init // 将当前目录加入版本管理

  • git pull // 同步代码

  • git add . // 添加所有修改到缓存区(注意：这里是修改，不是修改文件)

  • git commit -m ‘xxx’ // 添加到head
  • git commit -v // 提交时显示所有diff信息
  • git push origin master -f // 可以把 master 换成你想要推送的任何分支 -f是强制推送

  • git remote add origin // 添加远征目录

  • git switch -c dev // 从当前版本创建新并切换dev分支

  • git switch master // 切换到已创建的master分支

  • git branch -d dev // 删除分支

  • git branch // 查看分支

  • git branch // 创建分支

  • git push origin dev // 将分支推到远程

  • git diff master dev // 合并前查看文件差异

  • git merge dev // 当前是master，将dev合并到当前分支 （合并后再add和commit）

  • git reset –hard origin/master

  • git stash // 暂存改变，还原工作目录

  • git stash pop // 取回暂存

  • git rm -r –cached <文件> #删除追踪状态

安装更新git版本

brew install git
// 安装完后
brew link git --overwrite // 改变默认git指向
which git // Linux which命令用于查找文件
git --version // 查看版本
安装/更新衍生的问题

Updating Homebrew 等很久时，按住 control + c 之后命令行会显示 ^C，就代表已经取消了 Updating Homebrew 操作
大概不到 1 秒钟之后就会去执行我们真正需要的安装操作了

变成中文:在~/.bash_profile文件中添加 export LANG=en_US.UTF-8 即可

进阶

三个区: 工作区(work directory)、stage(暂存区)、版本库(repository)

版本回退

用HEAD表示当前版本(最新的提交)

上一个版本就是HEAD^，上上一个版本就是HEAD^^

往上100个版本写成HEAD~100

git reset --hard HEAD^　// 第一种写法 HEAD^
git reset --hard commit_id　// 第二种写法 这里的版本号正常是32位的hash。这里可以只写前几位，git会自动识别补全

git log 可以查看提交历史，以便确定要回退到哪个版本。
git reflog 查看命令历史，以便确定要回到未来的哪个版本
修改管理
git checkout -- readme.txt

意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

总之，就是让这个文件回到最近一次git commit或git add时的状态。

git checkout – file命令中的–很重要，没有–，就变成了“切换到另一个分支”的命令

用命git reset HEAD 可以把暂存区的修改撤销掉（unstage），重新放回工作区。

git reset命令既可以回退版本，也可以把暂存区的修改回退到工作区。

藏匿工作区(stash)
git stash // 藏匿
git stash list // 藏匿记录列表
git stash pop // 恢复最近一次藏匿的工作区，并将记录删除
git stash drop // 删除stash
git stash apply stash@{0} // 多次stash，恢复的时候，先用git stash list查看，然后恢复指定的stash

远程分支版本回退

git reset --hard HEAD~1 
git add .
git commit -m '回到上一提交'
git push -f // 强推到远程分支

reset 三种模式

  • soft 不清除stage
  • mixed 清除stage 不清除work directory
  • hard 清除stage 清除work directory
